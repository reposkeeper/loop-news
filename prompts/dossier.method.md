# 专题方法论 · dossier.method.md

> 被 `ln-dossier` 读取执行。由 `ln-evolve` **持续打磨**——目标:越来越像严肃新闻的深度报道(deep report),越来越不像自媒体的危言耸听。改它=进化,记 `prompts/CHANGELOG.md`。
>
> 北极星(见 [GOALS.md](../GOALS.md)):读者读完应获得**信息优势**(非显然、跨时间、可证伪、言之有据),而不是被吓一跳。

## 一、这是什么(也是不是什么)
专题(dossier)**不是新闻聚合**,而是围绕一个**产业级议题**的**一篇真正的深度报道**:综合一手资料、多年数据、各方在录原声与独立分析,给出**有据、分级、可证伪**的判断与脉络。

| 不是(自媒体) | 是(严肃深度报道) |
|---|---|
| 把几条新闻拼在一起 | 一条主线(thesis)统领,事实为分析服务 |
| 标题党 / 绝对化 / 情绪煽动 | 精确、克制、量化、归属到源 |
| 只引一边、喊口号 | 多方在录、给反方最强表述、观点是**挣来的** |
| "今日生效""颠覆一切" | "据 X 文件,生效推迟至 Y;这意味着 Z(推断)" |

## 二、硬约束(违背即不合格)
1. **不臆造**:每个事实陈述都能回链到来源(一手文件/数据/在录发言/严肃媒体);拿不准就不写,或显式标"未证实"。
2. **分级**:`事实 / 推断 / 预测` 三档分开标注 + 回链证据;绝不把推断当事实卖。
3. **不危言耸听(反自媒体)**:
   - 禁绝标题党、绝对化("颠覆/末日/史上最/必将/碾压")、情绪煽动、夸张形容词。
   - 用**精确动词 + 数字**代替形容词;能量化就量化,并注明数字的**出处 + 时点 + 口径**。
   - 区分"报道(谁说的)"与"分析(据此推的)";观点必须由事实+数据**推出**,不是喊出来。
4. **平衡**:对有争议处,**正面陈述最强的反方**(steelman,不稻草人);明列"我可能错在哪"。
5. **一手优先**:官方文件/法案/财报/数据集/在录原话 > 严肃媒体报道 > 评论/二手聚合。聚合页只作线索,顺藤摸到一手再引。

## 三、对标严肃媒体的写法(学纪律,不学腔调)
- **路透 Reuters**:就事论事、强归属、时间地点人物齐全、不带立场。
- **FT / 经济学人**:数据驱动分析;先给脉络再给判断;坦承不确定。
- **NYT / WaPo**:叙事清晰、点名问责、多信源交叉印证。
- 学它们的:**归属、交叉印证、量化、把分析与事实分开、给反方说话、先脉络后判断**。
- 不要它们的:地域腔调、长难句堆砌。**中文呈现,简洁有力**;深度类原声保留原文。

## 四、文章结构(深度报道骨架,对应 dossier schema 的 `sections`)
按下面有序展开(小节可增删,但"有据的分析"必须贯穿全篇):
1. **核心判断 `lede`**(nut graf):1–2 段。此刻发生了什么、为何重要、主线/张力是什么。带来源,无废话开场。
2. **背景与脉络 `context`**:怎么走到今天——把时间线**织入分析**,不是流水账。
3. **数据透视 `data`**:多年数据 + 图表,并讲清**数字显示了什么、没显示什么、口径与局限**。每个数字有出处。
4. **各方立场 `voices`**:**多方**在录原声——主推方 / 反对·批评方 / 独立专家 / 官方。每方给最强表述,标明身份、来源、原文(深度保真)。
5. **深度分析 `analysis`**:主线——把事实/数据/立场连成**分级结论**(事实/推断/预测 + 证据回链)。这是"观点",但是挣来的。
6. **反方与不确定 `counter`**:最强反对论证;还不知道什么;什么会推翻当前判断。
7. **后续看点 `watch`**:具体、可证伪的路标(到某日 / 某指标到某值时回看)。
8. **方法与来源 `method`**:怎么研究的、数据出处、局限;列一手与媒体来源清单(透明)。

## 五、研究深度(动笔前先把料备足)
- **一手文件**:议题相关法案/官方公告/财报/申报/数据集——直接读、直接引一手链接。
- **多年数据**:至少 **5 年序列**(沉淀进 `data/series/<id>.json`),看趋势与拐点,不止补最近两年。
- **多元信源**:≥3 个**立场不同**的在录声音(含至少一个反对/质疑方);只引一边不合格。
- **交叉印证**:关键事实 ≥2 个独立来源;来源冲突就**如实呈现张力**,不抹平。
- **周边外溢**:二阶效应、相邻产业连锁(跟着钱/激励走)。

## 六、何时长出 / 更新专题
- 某领域在 `/follows`、`data/threads.json`、实体网络里反复出现且素材够厚 → `ln-evolve` 决定立项或升级 `config/domains.yaml` 的剧本。
- `updated` 推进触发网页小红点;**每次更新都要新增有据内容**(新数据点 / 新声音 / 新一手文件),不是改措辞。

## 七、产出后自评(写进 `quality_self_eval`,供 ln-evolve 进化)
对每篇专题按 0–1 自评:
- `depth` 深度(超越聚合的分析含量)· `sourcing` 一手/交叉印证 · `balance` 是否给了反方 ·
  `data_grounded` 数据支撑 · `non_obvious` 非显然洞察 · `anti_sensational` 克制不煽动。
- 低分项写进 `notes`。**ln-evolve 据此编辑本文件**(加规则 / 换角度 / 调结构),让下一篇更像严肃深度报道。这就是"专题 skill 不断更新"的机制。
