# rubric.md — 个人 8 维评价口径 · base

> **承 [`prompts/scoring.md`](../../prompts/scoring.md) 的八维框架**,把同一套尺子用在**这个人那片**(其订阅话题条目 + 其反馈 + 其画像变更)上 = **个人进化分数**。`composite = 八者均值`;进化能力 = 成熟度等级。**明确不含系统迭代分**(§8.4)。
>
> **可见性硬约束(§8.5):真实评分框架 = 系统核心,任何用户不可见。** 本文件、8 个维度名、权重、真实分**都不下发客户端、无 API 返回**;用户端只见服务端翻译出的**不透明等级 + 进度 + 会变叙事**(让他「感到越来越懂我」)。真实分只在 owner 侧可见。本 rubric 是**内部评价口径**,给通道 B 的 runner 当上下文,不是给用户看的。

## 八个维度(个人尺度,各 0–100)
沿用全局的八维语义,over「这个人那片」重新解释:
1. **关联度**:为他连出的跨时间/跨域关联有多深(他此前得不出的)。
2. **数量**:他视野里够不够料(过度折叠/过窄订阅会失血——见第 8 维兜底)。
3. **分析整合**:呈现给他的结论质量、证据回链率。
4. **广度**:他的镜头覆盖面在不在合理地扩(不是越窄越好)。
5. **来源多样**:为他选源的质量与**不单一**(与第 8 维协同,防只喂一个圈子)。
6. **及时性**:他看到的是当日一手还是旧闻。
7. **克制 restraint(个人化 = 反过拟合 / 反回音室)**。
8. **创新 innovation(个人化 = 反过滤气泡)**。

## 第 7 维 · 个人克制 = 反回音室(重点)
懂这个人不等于**过拟合**他的历史点击。个人克制维在下列情形扣分:
- **`muted` 过猛 / 只留一种声音** → 把他关进回音室 → 克制↓。
- 为讨好其偏好而**放松证据分级**、堆「投其所好但无据」的关联 → `grounded`↓(承 scoring.md 的 `overreach_rate`)→ 克制↓。
- 信噪比:呈现给他的条目没被结论真正引用(水文尾巴)→ `snr`↓ → 克制↓。
- **本质**:个人克制 = 承 GOALS「宁缺毋滥 / 不臆造」在个人尺度的运行化——**宁可少推一条,也不喂气泡回音**。

## 第 8 维 · 个人创新 = 反过滤气泡(重点)
防画像**收敛成过滤气泡**、防「只喂他最近爱看的」高分停滞。扣分情形:
- 兴趣**过窄**、始终只吃他固化偏好的话题/来源 → `exploration`↓ → 创新↓。
- 从不给他 serendipity(共识缺口 / 逆共识 / 跨域惊喜)→ `new_productive` / `cross_domain_novel`↓ → 创新↓。
- **本质**:个人创新 = 天然的**过滤气泡护栏**——始终为他留 serendipity 名额。

## 张力矩阵(个人版 · 抄近路必被另一维拖回)
| 抄近路(过度个性化) | 涨 | 跌(净不赚) |
|---|---|---|
| 过度 mute、只留一种声音 | 贴合度↑ | 个人克制↓(回音室) |
| 只喂他最近爱看的、兴趣越收越窄 | 短期贴合↑ | 个人创新↓(过滤气泡) |
| 为投其所好放松证据/堆无据关联 | 关联↑ | 个人克制↓ |
| 为求新乱推噪音讨好 | 创新↑ | 个人克制↓ |

`composite = 八者均值` + 每轮**提最低分**(见 `loop.md`)⇒ 收敛到**既懂他、又不把他关进气泡**。**输出永远只给翻译版,真实分留内部/owner。**
