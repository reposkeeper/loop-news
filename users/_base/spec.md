# spec.md — 个人筛 / 排 / 强调 / 折叠规范 · base

> 对全局采集/汇总方法论(`prompts/synthesize.method.md` 的八个透镜)的**个人化补充**:同一份共享语料,怎么按**这个人**的镜头呈现。
> base 版 = 中性缺省(≡ owner 底座);分叉后通道 B 只改「这个人」相关的少数条目。**规范只影响排序/强调/折叠,不改语料本身,也不改结论的证据分级**(分级永远承全局纪律)。

## 一、筛(选什么进这个人的视野)
- **默认全收 base 池**:新用户订阅 ⊆ base 话题,静态页已含全部 base 条目 → 个人化只是**重排 + 折叠 + 「为你」标签**,零额外载荷(热路径见 §4.4)。
- **订阅优先**:`profile.topics` / `profile.entities` 命中的条目上浮;长期零互动的话题**折叠**(不删除,留可展开)。
- **`muted` 硬隐藏**:进了 `muted` 的话题/实体/来源不出现在主视野——但受克制维约束:**mute 过猛会触发反回音室**(见 `rubric.md`),此时保留少量对立面「透气口」。

## 二、排(这个人先看到什么)
优先级从高到低,确定性可解释:
1. **对他非显然**:跨时间/跨域、且他此前未必接触过的深层判断(承 `goals.md`)。
2. **共识缺口 / 逆共识**:主流叙事 ≠ 圈内原声之处——最高价值信号,即便不完全贴其兴趣也留位(反过滤气泡)。
3. **画像权重**:`topics`/`entities`/`sources`/`tones` 加权;`tones` = 深度 / 共识 / 数据 / 原声 四类语气的个人偏好。
4. **及时性与线索**:当日一手 > 旧闻;正在追的线索/专题续集置顶。

## 三、强调(怎么让重点跳出来)
- 对这个人**最非显然**的一两条,给「为你」不透明标签(**不下发任何权重/维度/分数**,§8.5 硬约束)。
- 沿用全局呈现纪律:每块正文最多 2 处高亮;深度类保留原文原声,不改写丢锋芒。

## 四、折叠(怎么省他的注意力)
- 低权重话题、已读、共识类重复事件 → 折叠成一行摘要,可展开。
- 水文尾巴(未被任何结论真正引用的条目)→ 降到底部或折叠(信噪比,呼应 `rubric.md` 克制维的 `snr`)。
- **折叠 ≠ 删除**:随时可展开;画像变化后自动回涨。

## 五、边界(这份规范不做什么)
- 不改② 共享语料、不改证据分级、不臆造关联(硬约束承 GOALS)。
- 只表达**这个人**的镜头;不外溢为全局规范(全局归 `prompts/*.md` + `ln-evolve`)。
